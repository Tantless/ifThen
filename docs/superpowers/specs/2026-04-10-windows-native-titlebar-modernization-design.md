# Windows 原生标题栏现代化设计

日期：2026-04-10  
范围：`desktop/` 桌面端  
状态：待评审

## 1. 背景

当前桌面端使用：

- Electron frameless 窗口
- `titleBarStyle: 'hidden'`
- React 自定义标题栏 `desktop/src/frontui/WindowTitleBar.tsx`

现状已经具备“自绘标题栏”的技术基础，但视觉上仍偏实色、偏硬边：

- 窗口背景为固定实色
- 标题栏是纯色块
- 窗口外轮廓现代感不足
- 与 Windows 10 / 11 的原生应用气质仍有距离

用户目标是：

1. 尽量靠近 Windows 原生桌面应用观感
2. Win10 / Win11 都友好
3. 保留当前自定义标题栏结构
4. 主体页面尽量不大改
5. 用小圆角替代硬尖角，提升现代感

## 2. 已确认约束

- 兼容目标：**Win10 + Win11 都友好**
- 实现边界：**保留现有自定义标题栏结构**
- 影响范围：**只改窗口外轮廓 + 标题栏**
- 降级策略：**Win11 优先更强效果，Win10 自动降级**
- 风格方向：**保留当前产品基调，只做更精致的现代化升级**

## 3. 目标

### 3.1 视觉目标

- 标题栏从纯实色升级为更接近 Windows 原生气质的浅色现代标题栏
- 窗口边缘采用小圆角
- Win11 下获得更强的“原生桌面应用感”
- Win10 下不追求完全一致，但必须自然、稳定、不违和

### 3.2 工程目标

- 保持 frameless + 自定义 drag region + 自定义控制按钮方案
- 不重构主内容三栏布局
- 不改动主要交互流程
- 不引入大面积设计系统重写

## 4. 非目标

本次明确不做：

- 重做主聊天区、侧边栏、分析面板的整体视觉体系
- 大规模圆角化整个应用内部组件
- 改成系统原生标题栏组件
- 新增消息发送、工具栏功能、系统菜单等新业务功能
- 为 Win10 强行模拟和 Win11 完全一致的原生材质

## 5. 方案选择

最终采用：

## **保留现有自定义标题栏结构 + 接入 Windows 友好的原生窗口质感增强**

原因：

- 比纯 CSS 拟态更接近“系统原生”
- 比改窗口 chrome 结构更稳
- 最符合“少改主体 + 更现代 + 更 Windows 友好”的目标

## 6. 设计方案

### 6.1 Electron 窗口层

文件：

- `desktop/electron/main.ts`

设计原则：

- 保留 `frame: false`
- 保留 `titleBarStyle: 'hidden'`
- 继续由 React 渲染标题栏
- 在窗口层增加“Windows 友好外观增强配置”

目标效果：

- **Win11**：优先启用更现代的系统窗口材质/圆角能力
- **Win10**：走保守降级，不依赖同等级材质能力

窗口层预计改动方向：

1. 调整窗口背景策略，避免当前硬实色背景破坏玻璃感
2. 为 Windows 平台增加条件化的现代窗口外观配置
3. 确保 frameless 前提下窗口边缘裁切与圆角表现干净

### 6.2 标题栏组件层

文件：

- `desktop/src/frontui/WindowTitleBar.tsx`

保留：

- 当前标题栏布局结构
- 左侧标题文本
- 右侧三个窗口控制按钮
- `WebkitAppRegion: 'drag' / 'no-drag'`

升级内容：

1. 标题栏背景改为更轻的半透明浅色层
2. 增加极轻的底边 / 分隔线
3. 优化按钮 hover / active 反馈，更接近 Windows 交互习惯
4. 顶部与窗口整体圆角视觉保持一致

不做：

- 改控制按钮布局
- 改按钮语义
- 改成 macOS 风格或浏览器标签页风格

### 6.3 CSS / 壳层层

文件：

- `desktop/src/styles/frontui.css`
- 必要时少量补充 `desktop/src/styles.css`

改动目标：

1. 给桌面根容器与标题栏建立正确的裁切关系
2. 保证圆角只体现在窗口外轮廓，不影响主体布局结构
3. 让标题栏与主内容衔接自然
4. 避免圆角与透明背景造成边缘露底、锯齿、深色缝隙

### 6.4 Win10 / Win11 分层策略

#### Win11

- 优先获得更强的原生现代感
- 标题栏允许更明显的浅色半透明感
- 圆角表现更接近系统默认审美

#### Win10

- 不强求与 Win11 一样的系统材质
- 使用更稳妥的浅色半透明 / 轻层次策略
- 重点保证稳定和协调

换言之：

- **Win11 更高级**
- **Win10 不破相**

这属于有意识的“平台差异化增强”，不是缺陷。

2. 标题栏比当前明显更现代，但仍保留原产品基调

## 10. 风险

### 风险 1：Electron 窗口层能力在不同 Windows 版本上表现不一致

应对：

- 用条件化策略区分 Win10 / Win11
- 先确保稳定，再增强效果

### 风险 2：frameless + 圆角 + 半透明组合导致边缘瑕疵

应对：

- 把圆角控制集中在窗口根层与标题栏层
- 避免主体页面大量参与透明叠加

### 风险 3：开发态与打包态视觉存在差异

应对：

- 除开发环境外，需做打包后实机验证

## 11. 实施原则

本次实现应遵循：

1. 先做窗口层基础能力
2. 再做标题栏视觉增强
3. 最后微调 hover、边框、圆角衔接
4. 不顺手扩展到主体页面改版
