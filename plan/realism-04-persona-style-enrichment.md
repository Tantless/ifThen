# TODO 04：人格与表达风格增强

## 问题

现有 persona 已经总结双方稳定互动倾向，但仍偏抽象。真实聊天的拟真性很大程度来自更细的表达风格：短句比例、连续发送习惯、语气词、标点、回应间隔、冲突时是否收缩等。

这些信息很多可以从消息记录中确定性统计，不需要额外 LLM 调用。

## 目标

在不显著增加完整分析耗时的前提下，补充可执行的 style profile，让推演和实时会话更像真实对方。

## 风格维度

- 消息长度：平均长度、中位长度、长消息比例。
- 短句习惯：是否经常连续发多条短消息。
- 标点习惯：逗号、句号、问号、感叹号、省略号、波浪线使用倾向。
- 语气词：如 hhh、哈哈、嗯、哦、好、啊、吧、嘛、呀。
- 回复密度：一次回应通常是一条还是多条。
- 推进习惯：是否主动问问题、是否常接话、是否常转移话题。
- 冲突习惯：压力下变短、解释、回避、反问、冷处理、给台阶。
- 延迟习惯：可粗略统计不同场景下的回复间隔桶。

## 数据存储策略

第一版建议不新增表，优先在推演时从已加载 messages 中按需计算或缓存到 context pack snapshot。后续如果性能或复用需要，再考虑持久化。

## 验收标准

- [ ] style profile 不需要额外 LLM 调用。
- [ ] style profile 能区分 self 和 other。
- [ ] simulation prompt 能收到 other 的表达风格约束。
- [ ] 回复长度和短句习惯更接近样例中的对方。
- [ ] 不因为风格统计引入明显分析耗时增长。

## 风险

- 统计过细可能把临时状态误当成长期风格。
- 表情、图片、语音等非文本消息可能影响统计，需要明确是否忽略。
- 中文、英文、表情符号混合文本的长度统计需要保守处理。
