# 项目待办事项管理

本文件夹包含项目的所有待办事项和计划任务。

## 文件说明

- **current-tasks.md** - 当前正在进行或即将开始的任务
- **backlog.md** - 待排期的功能和优化项
- **completed.md** - 已完成的任务归档

## 任务优先级

- **P0** - 阻塞性问题，必须立即处理
- **P1** - 高优先级，影响核心功能
- **P2** - 中优先级，重要但不紧急
- **P3** - 低优先级，优化和增强

## 任务状态

- **TODO** - 待开始
- **IN_PROGRESS** - 进行中
- **BLOCKED** - 被阻塞
- **REVIEW** - 待审查
- **DONE** - 已完成

## 使用规范

1. **开始新任务前**：查看 `current-tasks.md`，选择优先级最高的任务
2. **任务进行中**：更新任务状态为 `IN_PROGRESS`，记录进展
3. **任务完成后**：将任务移至 `completed.md`，更新完成日期
4. **发现新需求**：添加到 `backlog.md`，标注优先级

## 协作约定

- 每个任务必须有明确的验收标准
- 任务完成后必须通过测试验证
- 重大变更需要更新相关文档（CLAUDE.md、AGENTS.md）
- 定期（每周）review backlog，调整优先级
